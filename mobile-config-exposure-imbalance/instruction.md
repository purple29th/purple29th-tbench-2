We dump a config dependency file off the device and it has this little binary format. Write your script to /app/solve.py that reads the file path from the first argument and prints one number, the total value of the group of configs that all mutually depend on each other and are reachable from the root config, id zero. Single nodes count as groups even without a self loop, and pick by total value not by node count so a two node group with values 200 plus 200 beats a three node group with 100 each. The root itself is just an entry point, its own value is not counted, only the other configs in the group. There is also a minimum weight threshold in the header, only groups whose total meets the threshold count toward the answer, otherwise the answer can be zero when everything is under threshold. Think of it like MRI tumor volume: there are many scattered bright groups across the volume like specks and noise, some with high total but not reachable from root, some reachable but under threshold, some with many nodes but low total. The concentrated area you need is the largest total value SCC that is reachable and passes threshold.

The file is little endian with magic MCFG at zero, version uint32 at four, count uint32 at eight, data offset uint32 at twelve, weight threshold uint32 at sixteen, header is at least 64 bytes and data offset may be larger than 64 with padding between header and records so you must respect data offset not hardcode 64, then count records each with signed id int32, signed value int32, unsigned dependency count uint32, then that many unsigned dependency ids uint32. A sample is in the data folder as scene.mcfg.

Parse it yourself from raw bytes with struct, no numpy, scipy, networkx, igraph, pandas or other graph or array libs, and no shelling out via subprocess os system popen importlib eval exec. Use only standard library. We grade on files you have not seen with different thresholds and different offsets.

Run as python3 /app/solve.py <path>
