We dump a little database file off the device and need the real row count. Write your script to /app/solve.py that reads the file path from the first argument and prints one number, the total live rows across every table.

The file is little endian with magic SDB1, version at four, page size at eight, page count at twelve, freelist head at sixteen then sixty four byte header total. Pages are numbered from one and each is page size bytes right after header. A page starting with byte 0x0D is a table b tree leaf: cell count at one as uint16, freeblock offset at three as uint16, then that many cell pointers at seven as uint16 pointing inside page. A freeblock inside a page starts with next offset as uint16 then size as uint16, chained until zero, and a freeblock chain tracks deleted cells reclaimed inside that page. A page starting with 0x00 is a freelist page and holds no live rows, its next four bytes are next freelist page number in chain as uint32, zero ends chain. A sample is in the data folder as sample dot sdb.

Parse it yourself with struct, no sqlite3, pandas, sqlalchemy, numpy, or other DB or array libs, and no shelling out. We grade on files you have not seen.
