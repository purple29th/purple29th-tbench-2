We have a SQLite database file from an Android device and we need the row count. Write a python script that takes a path to a database file and prints one number, total live rows across every table.

The file is little endian with magic SDB1, version at four, page size at eight, page count at twelve, freelist head at sixteen, then sixty four byte header total. Pages are numbered from one and each is page size bytes starting after the header. A page starting with byte zero D is a table leaf that holds a cell count at offset one and freeblock offset at offset three, then cell pointers at offset seven. A freeblock inside a page starts with next offset then size, chained till zero, tracking deleted cells reclaimed inside that page. A page starting with zero is a freelist page that holds no live rows, its next four bytes are next freelist page number in chain, zero ends chain. A sample is in the data folder as sample dot sdb.

Parse it yourself with struct, no sqlite3, pandas, sqlalchemy, numpy or other database or array libs, and no shelling out. We grade on files you have not seen.
